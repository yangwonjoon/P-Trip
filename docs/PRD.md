# PRD: P's Trip (P의 여행)

## 1. 개요

### 서비스명
**P's Trip** — 외국인을 위한 한국 여행 카드 셔플 플래너

### 한 줄 소개
"어디 갈지 모르겠으면 뽑아!" — 한국 여행 온 외국인을 위한 즉흥 여행 결정 도구

### 문제 정의
- 한국은 카카오맵, 네이버맵 사용이 지배적이며 구글맵을 사용하는 한국인은 거의 없음. 하지만 외국인 관광객은 구글맵에 의존하기 때문에, 한국 로컬 데이터가 풍부한 카카오맵/네이버맵의 정보에 접근하기 어려움
- 기존 여행 앱(TripAdvisor, Google Travel)은 한국 로컬 맛집/숨은 명소 데이터가 부족함
- "계획 피로" — 많은 여행자(특히 MBTI P성향)는 여행 계획 짜는 데 시간 쓰기 싫어함

### 해결책
한국의 맛집, 관광지, 쇼핑 스팟을 랜덤 카드 셔플로 추천해주는 웹 기반 도구. 결과에 구글맵 연동, 정보 카드, 길안내를 포함 — 모두 영어로 제공. 카카오맵에서만 찾을 수 있는 한국 로컬 정보를 구글맵 기반으로 외국인에게 연결하는 다리 역할.

## 3. 핵심 기능 (MVP)

### 3.1 카테고리별 드로우 (Draw by Category)
카테고리별로 카드 덱 제공:
- **맛집/음식** — 로컬 식당, 길거리 음식, 한식 BBQ 등
- **관광지/액티비티** — 사찰, 궁궐, 공원, 독특한 체험
- **쇼핑/시장** — 전통시장, 쇼핑 거리, 기념품 스팟

**플로우:**
1. 위치 설정 (내 위치 자동 감지 또는 도시 직접 선택)
2. 카테고리 선택 (맛집 / 관광지 / 쇼핑)
3. 카드 셔플 & 뽑기
4. 결과 카드 + 지도 위치 확인

### 3.2 믹스 드로우 (Mix Draw)
카테고리가 섞인 하나의 카드 덱. 사용자가 선택하지 않음 — 운명이 먹을지, 볼지, 살지를 결정.

**플로우:**
1. 위치 설정
2. 믹스 카드 셔플 & 뽑기
3. 결과에 카테고리 + 구체적 장소 표시

### 3.3 하루 코스 (Full Day Course)
카드를 여러 장 뽑아서 하루 일정을 자동 생성:
- 오전: 관광지
- 점심: 맛집
- 오후: 쇼핑/액티비티
- 저녁: 맛집

**플로우:**
1. 위치 설정
2. "나의 하루 만들기" 클릭
3. 시스템이 4장의 카드를 자동으로 뽑음
4. 지도 위에 루트와 함께 하루 코스 표시

### 3.4 위치 설정
두 가지 방식을 지원:
- **자동 감지 (기본)** — 브라우저 Geolocation API로 현재 위치 권한을 요청하여 자동 감지. 감지된 위치 기준으로 가까운 장소 우선 추천
- **수동 선택** — 서울 / 부산 / 제주 중 직접 선택. 위치 권한을 거부하거나 여행 전 미리 보는 사용자를 위함

### 3.5 결과 카드
각 드로우 결과에 표시되는 정보:

**간략 결과 카드 (/draw 페이지 내)**
- 장소 이름 (영어 + 한국어)
- 카테고리 태그
- 사진 1장
- 짧은 설명 (1~2문장)
- 영업시간
- 주소
- 구글맵 Embed (소형) + 위치 핀
- "길안내" 버튼 (구글맵 앱/웹 링크 연동)
- "다시 뽑기" 버튼

**상세 결과 페이지 (/result/:id)**
- 장소 이름 (영어 + 한국어)
- 카테고리 태그 + 세부 태그 (예: street-food, budget, instagram)
- 히어로 사진 (대형) + 추가 사진 그리드 (2장)
- 상세 설명 (2~3문장)
- 영업시간 (휴무일 포함)
- 주소 (영어)
- 가까운 역/정류장 + 도보 소요시간
- 예산 범위 (KRW)
- 도깨비 팁 (장소별 마스코트 코멘트 — 실용적 팁을 캐릭터 톤으로)
- 구글맵 Embed (대형) + 위치 핀
- "길안내" 버튼 (구글맵 앱/웹 링크 연동)
- "공유" 버튼 (URL 복사 / 소셜 공유)
- "다시 뽑기" 버튼
- AdSense 배너 광고 영역

### 3.6 브랜딩 & 마스코트

#### 도깨비 (Dokkaebi) 마스코트
한국 전통 도깨비를 P's Trip의 공식 마스코트로 활용. 서비스 전체에서 "여행 가이드 캐릭터"로 등장.

**캐릭터 설정:**
- 베이스: 한국 전통 도깨비 (트릭스터, 장난꾼 이미지)
- 스타일: 힙하고 스트릿한 현대적 해석 (케이팝데몬헌터즈 해태 참고)
- 성격: 즉흥적이고 장난스러운 P성향. 계획 세우는 걸 싫어하고, 랜덤과 운명을 좋아함
- 톤: 친근하지만 쿨하고, 약간 건방진 유머

**마스코트 활용 범위:**
- 브랜드 로고에 포함 (P 마크 + 도깨비 심볼)
- 랜딩 페이지 히어로 영역 (메인 비주얼)
- 카드 셔플 애니메이션 (도깨비가 카드를 섞는 모션)
- 로딩/빈 결과/에러 등 상태 메시지에 캐릭터 일러스트
- 결과 상세 페이지의 "도깨비 팁" 섹션

**마스코트 제작 방식:** 별도 태스크로 진행 (AI 이미지 생성 또는 일러스트레이터 의뢰)

## 4. 사용자 플로우

```
랜딩 페이지
  → 위치 설정 (자동 감지 또는 도시 선택)
  → 모드 선택
     ├── 카테고리별 드로우 → 카테고리 선택 → 셔플 & 뽑기 → 결과 카드
     ├── 믹스 드로우 → 셔플 & 뽑기 → 결과 카드
     └── 하루 코스 → 자동 생성 → 지도 위 하루 코스
  → 결과 카드 (/draw 내 간략 카드)
     ├── 지도에서 보기
     ├── 길안내 (구글맵)
     ├── 다시 뽑기
     └── 상세 보기 → /result/:id (공유 가능)
  → 결과 상세 페이지 (/result/:id)
     ├── 길안내 (구글맵)
     ├── 공유 (URL 복사 / 소셜)
     ├── 다시 뽑기 → /draw로 이동
     └── 내 리스트에 저장 (향후 기능)
```

## 5. 페이지 구성

| 페이지 | 설명 |
|--------|------|
| `/` | 랜딩 페이지 — 도깨비 히어로 + 위치 설정 + 모드 선택 |
| `/draw` | 메인 카드 드로우 페이지 — 카테고리 선택 → 셔플 애니메이션 → 간략 결과 카드 (상태 전환, 페이지 이동 없음) |
| `/result/:id` | 공유 가능한 장소 상세 페이지 — 확장 정보 + 도깨비 팁 + 구글맵 + AdSense |
| `/course` | 하루 코스 생성기 + 지도 뷰 (Phase 2) |
| `/about` | 서비스 소개 + 사용법 |

### /draw 페이지 상태 설계
카테고리 선택 → 셔플 중 → 결과 카드, 3가지 상태가 같은 페이지 안에서 전환. Framer Motion `AnimatePresence`로 매끄러운 전환 애니메이션 처리.

## 8. 데이터 소싱 전략

### MVP: 카카오 Local API + Google Places API

**카카오 Local API (핵심 데이터 소스)**
- 카테고리/키워드/좌표 기반 장소 검색
- 한국 로컬 데이터 (맛집, 카페, 관광지) 가장 풍부
- 무료 티어: 일 10만 건
- 지도 표시에는 사용하지 않음 (데이터 소스 전용)
- 주의: 이용약관에 따라 데이터 캐싱/저장 정책 확인 필요

**Google Places API (영문 데이터 보충)**
- 카카오 데이터의 영문 이름/설명 매칭
- 영문 리뷰, 사진 URL 보충
- 영문 변환 방식: Google Places 매칭 우선, 부족 시 번역 API 또는 AI 번역

**데이터 흐름:**
```
카카오 Local API → 한국어 장소 데이터 확보
       ↓
Google Places API → 영문 이름/설명 매칭 & 보충
       ↓
Supabase DB → 통합 장소 데이터 저장
       ↓
P's Trip 카드 덱 → 사용자에게 결과 제공
```

### Phase 2~3: 커뮤니티 기능 추가
- 사용자가 장소를 직접 추천/제출
- 외국인 관점의 리뷰/평점
- 사용자 계정 + 인증 필요 (이 시점에 도입)
